KiddoAI: Building a RAG-based Learning Companion

Mar 15, 2025 • Achref Lajmi • Applied GenAI / Product Engineering

KiddoAI started as an academic MVP: the goal was to help children learn through interactive lessons that adapt to their pace. The core challenge was accuracy: we wanted the assistant to follow the official curriculum, not hallucinate content.

1) The Architecture

We used a pragmatic product architecture: a mobile client (Flutter), a backend API (Spring Boot), and a knowledge layer built from curriculum PDFs. The assistant retrieves relevant curriculum chunks and uses them to answer questions and generate lessons.

2) RAG Pipeline (Curriculum PDFs)

  • Split curriculum PDFs into small chunks with metadata (grade, subject, chapter).
  • Embed the chunks and store them for retrieval.
  • At runtime: retrieve top-k chunks and generate responses grounded in curriculum context.

3) Product Lessons Learned

  • UX matters: short, guided answers outperform long explanations for kids.
  • Guardrails: enforce “use retrieved context first” to reduce hallucinations.
  • Iterate fast: ship MVP flows early, then improve retrieval and prompts.

Tech Stack

Flutter • Spring Boot • PostgreSQL • LLM APIs • RAG • Voice (TTS/STT) • Git
